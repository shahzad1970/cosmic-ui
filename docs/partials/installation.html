Installation Cosmic UI requires no build step, no bundler, and no package manager. Add a single <script> tag and start building. Requirements Cosmic UI uses standard Web APIs that ship in every modern browser. Browser support: Chrome / Edge 90+, Firefox 90+, Safari 15.4+, and any Chromium-based browser. No polyfills needed. No IE support. Cosmic UI relies on Shadow DOM, Custom Elements v1, Constructable Stylesheets, and ES Modules — none of which are available in Internet Explorer. Download Clone the repository or download the source directly. There are no build artefacts — the src/ folder is the distribution.
# Clone the repo git clone https://github.com/user/cosmic-ui.git # Or download a zip curl -L https://github.com/user/cosmic-ui/archive/main.zip -o cosmic-ui.zip
If you prefer npm:
npm install @cosmic-ui/core
Quick Setup Import the library with a single <script type="module"> tag. All components register themselves automatically.
<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8" /> <meta name="viewport" content="width=device-width, initial-scale=1.0" /> <title>My App</title> <script type="module" src="path/to/cosmic-ui/src/index.js"></script> </head> <body> <ui-button background="indigo-600">Hello Cosmic!</ui-button> </body> </html>
That's it! No npm install, no webpack, no vite.config.js. Just a script tag and you're ready. Import Strategies Choose between importing the full library or individual components. Full Library Imports and registers every component. Simplest approach — recommended for most projects.
<!-- Registers all ui-* elements --> <script type="module" src="path/to/cosmic-ui/src/index.js"></script>
Individual Components Import only the components you need. Each file is self-contained and registers its own custom element on import.
<!-- Just the button --> <script type="module"> import './cosmic-ui/src/components/button/button.js'; </script> <!-- Or cherry-pick several --> <script type="module"> import './cosmic-ui/src/components/button/button.js'; import './cosmic-ui/src/components/input/input.js'; import './cosmic-ui/src/components/card/card.js'; </script>
JavaScript Imports You can also use standard ES module imports from JavaScript if you need access to the component classes or utilities.
import { UIButton, UICard, UIInput } from './cosmic-ui/src/index.js'; // Utilities are also available import { resolveColor, resolveSize } from './cosmic-ui/src/core/ui-utils.js';
Development Server Any static file server works. Cosmic UI ships as ES modules — no transpilation or bundling required.
# Using npx serve (included in project) npx serve . --listen 3000 # Or use any other static server python3 -m http.server 3000 php -S localhost:3000
Open http://localhost:3000 in your browser and you're good to go. Using with Frameworks Custom elements are a web standard — they work anywhere HTML works. Drop Cosmic UI components into React, Vue, Svelte, Angular, or plain HTML. React
// Import once at your app entry point import './cosmic-ui/src/index.js'; function App() { return ( <ui-card elevation="2"> <ui-card-body> <ui-text type="heading">Works in React!</ui-text> <ui-button background="indigo-600">Click me</ui-button> </ui-card-body> </ui-card> ); }
Vue
// main.js — import once import './cosmic-ui/src/index.js'; // vite.config.js — tell Vue to skip custom elements export default { plugins: [ vue({ template: { compilerOptions: { isCustomElement: (tag) => tag.startsWith('ui-'), }, }, }), ], };
Svelte
<!-- Import in your root layout or +page.svelte --> <script> import './cosmic-ui/src/index.js'; </script> <ui-button background="emerald-500">Svelte + Cosmic</ui-button>
